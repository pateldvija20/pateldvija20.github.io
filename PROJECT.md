# Portfolio ‚Äî Project Status

## What this is

A personal portfolio site built in Next.js 16 (App Router) with a heavily designed, interactive home scene. The design originates in Figma and is exported via Figma Make into React components living under `src/imports/`. The shell around those components ‚Äî interaction logic, animation, loading ‚Äî is hand-written.

## Shared vocabulary

These terms are used consistently throughout the codebase and in conversation.

**scene** ‚Äî The fixed-size `1440 √ó 1024 px` `div` that holds the home illustration. It is centered in the viewport and CSS-scaled down to fit smaller screens via `scale(min(100vw/1440, 100vh/1024))`.

**mat / cutting mat** ‚Äî The green grid cutting mat that sits in the center of the scene. It is the main surface the "objects" rest on. Also the name of the loading animation (see below).

**mat-grid** ‚Äî The `data-name` attribute value that identifies the mat DOM element inside the Figma import. Also used as the `LayerKey` for the mat in nav/interaction logic.

**layer / layer key** ‚Äî One of `book | file | folder | paper | mat-grid`. Each layer corresponds to one interactive object in the scene. The type is `LayerKey` in `HomeInteractive.tsx`.

**deck** ‚Äî The ordered stack of the four object layers (`book`, `folder`, `file`, `paper`). At any moment the first element is the "top" layer ‚Äî the one the user can interact with.

**cover overlay** ‚Äî A separate `div` positioned absolutely over the scene that renders a Figma-imported SVG component for an object. The Figma import renders a static version of each object; the cover overlay replaces that static visual with an interactive one (hover state, open state). There are three cover overlays: `bookCoverRef`, `fileCoverRef`, `folderCoverRef`.

**scatter** ‚Äî The animation where all four layers fly out from the deck to distinct positions across the scene, triggered by the first click or by clicking the Playground nav item. After scatter, all layers are individually clickable.

**restack** ‚Äî The reverse of scatter. Clicking the mat background when scattered returns all layers to their tilt-stacked positions.

**tilt** ‚Äî The default resting CSS `rotate()` transform for each layer when stacked (e.g. book: `rotate(-5.31deg)`). Values live in `LAYER_TILT`.

**flip / deck cycle** ‚Äî The scroll-driven animation. Scrolling down "flips" the top card off the deck (slides it up and off screen, teleports to bottom, animates in as the new bottom). Scrolling up reverses this. Five scroll segments cycle through all four layers.

**drag offset** ‚Äî Per-layer `{x, y}` translation accumulated from mouse drag. Stored in the module-level `dragOffsets` record (no re-renders). Composed with `baseTransform` when applying a transform to an element.

**base transform** ‚Äî The non-drag portion of a layer's transform string (tilt, scatter position, or selection scale). Stored in `baseTransformsRef`. Drag translate is prepended to it.

**import component** ‚Äî A TSX file under `src/imports/` generated by Figma Make. These are treated as **read-only static assets** ‚Äî never hand-edited. Interaction logic wraps them from outside.

**uUnroll** ‚Äî GLSL uniform in the cutting-mat shader. `0.0` = fully rolled cylinder, `1.0` = fully flat plane. GSAP drives it from 0 ‚Üí 1 over 3 seconds during the loading screen.

## What is implemented

### Loading screen (`src/components/Loading/`)

- **CuttingMat** ‚Äî WebGL canvas with a custom vertex shader. A `PlaneGeometry` is deformed: vertices to the right of `uUnroll` sit on a rolling semicylinder; those to the left lie flat. GSAP drives `uUnroll` 0 ‚Üí 1 over 3 s with a `power2.inOut` ease. Renders `/public/cutting-mat.svg` as its texture.
- **BackgroundPapers** ‚Äî Two decorative paper images (gray-blue bottom-left, olive-green top-right) shown behind the mat.
- **PercentageCounter** ‚Äî Displays the unroll progress as a percentage.
- **LoadingScreen** ‚Äî Orchestrator. Receives `onComplete`, fades the overlay out with GSAP after the mat finishes unrolling.

> **Status:** Fully implemented and functional. Loading screen is not currently wired into `page.tsx` ‚Äî `HomeInteractive` renders directly without waiting for load completion.

### Home scene (`src/components/HomeInteractive.tsx`)

- **Scene scaling** ‚Äî The `1440 √ó 1024` scene is pinned at center and scaled via CSS `transform` to always fit the viewport.
- **Sticky scroll container** ‚Äî `600vh` tall outer div with the scene `position: sticky` inside. Provides the scroll range for the deck cycle.
- **Deck cycle (scroll)** ‚Äî Scrolling through 5 segments cycles all four layers through the top position with anticipation + spring animations.
- **Scatter / restack** ‚Äî Click anywhere on the mat to scatter layers; click the mat background to restack.
- **Layer selection** ‚Äî After scatter, clicking a layer brings it to z=100 and scales it up. Nav items also trigger selection.
- **Drag** ‚Äî All four scattered layers are individually draggable. Drag offset is composed with base transform on every `mousemove` tick with no re-renders.
- **Reactive nav overlay** ‚Äî Replaces the static Figma nav. Shows `label -` indicator next to the active layer.
- **Book** ‚Äî Cover overlay: default state (`BookCover`), hover state (`BookHoverImg`), open state (`BookOpen`/Frame35). Click toggles open.
- **File** ‚Äî Cover overlay: default (`FileClosed`/Frame36), hover (`FileHoverImg`), open (`FilePurple`). Click toggles open.
- **Folder** ‚Äî Cover overlay: default (`FolderDefault`), hover (`FolderHover`), open (`FolderOpenView`). Click toggles open.
- **Paper** ‚Äî No cover overlay; uses the Figma import directly. No open state yet.

> **Status:** Core interactions are implemented. The scene renders correctly. Paper layer has no open/hover states. No page content exists behind the home scene (sections for About, Work, Notes, Resume, Playground are not yet built).

## Session change log ‚Äî HomeInteractive refactor (7 prompts)

### Prompt 1 ‚Äî Scroll-up animation fix ‚úÖ

**Problem:** `gsap.set()` and `gsap.to()` ran in the same frame on scroll-up, so the card snapped with no visible animation.

**Fix:** Wrapped the scroll-up `gsap.to()` (both `promotedEl` and `promotedCover`) in a double `requestAnimationFrame` so the browser commits the teleported `y: '-120%'` position to the screen before the tween begins.

Tween values confirmed: `y: 0`, `ease: 'power2.in'`, `duration: 0.5`, `onComplete: clearProps: 'transform'` on both.

---

### Prompt 2 ‚Äî Scroll ‚Üí Wheel event migration ‚úÖ

Removed the entire scroll-based architecture and replaced it with a wheel listener on the scene `div`.

| Removed | Added |
|---|---|
| 600vh outer div | `overflow: hidden` on outermost wrapper |
| `position: sticky` inner wrapper | (gone) |
| `window.addEventListener('scroll', ...)` | `scene.addEventListener('wheel', ..., { passive: false })` |
| `TOTAL_SCROLL`, `progress`, `rawSegment` | `isAnimatingRef` (blocks re-entry during tween) |
| `flippedCountRef` | `wheelEnabledRef` (starts `false`; gates the handler) |

- `e.preventDefault()` fires only when `wheelEnabledRef.current` is true.
- `isAnimatingRef` set `true` at tween start; set `false` in `onComplete` for both directions.
- `deltaY > 0` ‚Üí scroll-down (demote top card); `deltaY < 0` ‚Üí scroll-up (promote bottom card).

---

### Prompt 3 ‚Äî Touch support ‚úÖ

Added `touchstart` + `touchend` listeners to `sceneRef` inside the same wheel useEffect.

- `onTouchStart`: stores `e.touches[0].clientY`.
- `onTouchEnd`: threshold 30px. Swipe-up ‚Üí fires `onWheel({ deltaY: 1 })`; swipe-down ‚Üí `onWheel({ deltaY: -1 })`. Shares `isAnimatingRef` and `wheelEnabledRef` guards with the wheel path.
- Both listeners cleaned up in the effect's return function.

### Prompt 5 ‚Äî Mode toggle wired ‚úÖ

**New refs:** `wheelEnabledRef` (gates wheel/touch), `modeRef` (mirrors `mode` for closure access).

**`parseTiltDeg`** extracted from inside the wheel useEffect to module scope.

**`handleModeToggle` ‚Äî Scatter ‚Üí Stacked:**
1. Close all open overlays; reset `hasScatteredRef = false`, `selectedLayerRef = null`.
2. GSAP tween all 4 layers + covers: `{ x:0, y:0, rotation: tiltDeg, scale:1, ease:'power2.out', duration:0.6, delay: i*0.04s }`.
3. Set `wheelEnabledRef = true`, `modeRef = 'stacked'`, `setMode('stacked')`.

**`handleModeToggle` ‚Äî Stacked ‚Üí Scatter:**
1. Reset `selectedLayer`. Call `triggerScatter()`.
2. Set `wheelEnabledRef = false`, `modeRef = 'scatter'`, `setMode('scatter')`.

**`handleSceneClick`:** scatter only triggers when `modeRef === 'scatter' && !hasScatteredRef.current`; stacked mode now bails out (to be replaced in Prompt 6).

### Prompt 6 ‚Äî Click interactions + drag snap ‚úÖ

**New state/refs:**
- `selectedLayer: LayerKey | null` ‚Äî starts `null`.
- `selectedLayerRef` ‚Äî ref mirror for closure access.
- `lastMouseRef` ‚Äî tracks latest mouse position for drag-release zone check.

**New module-level helper:**
- `parseScatterTransform(t)` ‚Äî parses `translate(x, y) rotate(deg)` CSS string into `{ x, y, rotation }`.

**Extracted to stable component-level `useCallback`s:**
- `coverOf(layer)` ‚Äî maps layer key ‚Üí cover ref (was duplicated in 3 different effects).
- `syncZ(deck)` ‚Äî updates z-index + pointer-events for the 4-card deck (was duplicated in wheel useEffect and mode toggle).

**Stacked mode clicks (`handleSceneClick`):**
- Top card ‚Üí return early; existing DOM event listeners on book/file/folder handle open/close.
- Non-top card ‚Üí `isAnimatingRef = true`; anticipation tween `{ y: -20, duration: 0.15, ease: 'power2.out' }`; in `onComplete` teleport to `y: '-120%'` and run scroll-up promote path (syncZ, setActiveNav, double-rAF slide-in).

**Scatter mode clicks (`handleSceneClick`):**
- Mat click ‚Üí `reStack()`.
- Click already-selected card ‚Üí return; element listeners handle open/close.
- Click unselected card ‚Üí send previous card back to scatter pos (`power2.in`, 0.4s); fly new card to center `{ x:0, y:0, rotation:0, scale:1.04, power2.out, 0.5s }`. Updates `selectedLayerRef` + `setSelectedLayer`.

**Drag snap (`onMouseUp`, scatter mode only):**
- Desktop (‚â• 1024px): release within ¬±275 √ó `scaleRef.current` px of viewport centre X ‚Üí snap to centre.
- Mobile (< 1024px): release within 120px (2D) of viewport centre ‚Üí snap to centre.
- Snap to centre: GSAP `{ x:0, y:0, rotation:0, scale:1.04, power2.out, 0.5s }`, sets `selectedLayer`.
- Snap back: close open state first, then GSAP to scatter coordinates `{ power2.in, 0.4s }`. Clears `selectedLayer` if applicable.

## Current architecture state (after Prompt 7)

```
mode: 'scatter' | 'stacked'    useState, mirrors in modeRef
selectedLayer: LayerKey | null  useState, mirrors in selectedLayerRef ‚Äî starts null
wheelEnabledRef: boolean        false in scatter mode, true in stacked mode
isAnimatingRef: boolean         gate that blocks wheel/touch during deck transitions
deckRef: LayerKey[]             current deck order, index 0 = top card
```

**Event routing summary:**
- Wheel / touch ‚Üí deck cycle (stacked mode only, guarded by `wheelEnabledRef`)
- Scene click ‚Üí `handleSceneClick` (branches on `modeRef.current`)
- Drag release ‚Üí `onMouseUp` (snap logic active in scatter mode only)
- Nav click ‚Üí `handleNavClick`
- Mode button ‚Üí `handleModeToggle`
